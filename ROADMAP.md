# OpenIdentity Roadmap | خارطة طريق OpenIdentity

OpenIdentity is evolving from a practical manifest format into a portable discovery layer for AI agent identity.

## v0.1 — Draft Manifest Foundation

- Add AxiomID landing page, MVP, proof-of-concept, use-case, and agentic discovery documentation.
- Support bilingual English/Arabic positioning and light/dark mode product guidance.

- Define the first human-readable Markdown manifest shape.
- Provide JSON Schema validation for core structured fields.
- Document identity, controller, verification, capabilities, discovery, wallet, and authorization sections.
- Provide minimal, standard, and full examples.
- Align the repository structure for use inside AxiomID and compatible platforms.

## v0.2 — Interoperability Profiles

- Add MCP-specific tool registration guidance.
- Add A2A metadata examples and compatibility notes.
- Define signed manifest recommendations.
- Add domain and wallet verification recipes.
- Add machine-readable extraction conventions for Markdown manifests.
- Formalize `agentic.md`, `agentic.txt`, and optional `llms.txt` discovery conventions.

## v0.3 — Trust and Policy Layer

- Define authorization pointer profiles.
- Add policy references for delegated access.
- Add revocation and expiration guidance.
- Add controller transfer and recovery flows.
- Add stronger privacy guidance for memory discovery.

## v1.0 — Stable Portable Identity Descriptor

- Stabilize required and optional fields.
- Publish compatibility test fixtures.
- Define conformance levels for platforms.
- Support production adoption by AxiomID and other compatible agent platforms.

## Arabic Summary | ملخص عربي
